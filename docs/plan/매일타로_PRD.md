## 3. 핵심 기능 (MVP -- 3개)

### 기능 1: 오늘의 카드 뽑기

**설명:** 매일 1장의 타로 카드를 뽑고, 뒤집기 애니메이션과 함께 오늘의 메시지를 확인한다.

**상세 명세:**
- 앱 실행 시 홈 화면에 카드 뒷면이 표시됨
- 카드를 탭하면 3D 뒤집기 애니메이션(Y축 회전, 약 0.8초) 재생
- 뒤집기 완료 후 카드 앞면 + 카드 이름(한글) + 한 줄 메시지 표시
- 정방향/역방향 50% 확률 적용 (역방향일 때 카드 이미지 180도 회전)
- **일일 고정 로직**: 같은 날 재실행해도 동일한 카드 표시 (Hive에 날짜+카드ID+방향 저장)
- 카드 선택 알고리즘: `DateTime.now().day`를 시드로 한 의사난수 (서버 불필요)

### 기능 2: 카드 도감

**설명:** 78장 메이저/마이너 아르카나 전체 목록을 열람하고, 각 카드의 정방향/역방향 의미를 확인한다.

**상세 명세:**
- 그리드(2열) 형태로 78장 카드 썸네일 표시
- 탭별 필터: 전체 / 메이저 아르카나(22장) / 완드 / 컵 / 소드 / 펜타클
- 카드 탭 시 상세 화면으로 이동
- 상세 화면: 카드 이미지(크게) + 카드 이름 + 정방향 의미 + 역방향 의미
- 뽑은 적 있는 카드는 도감에서 별도 표시 (Hive에 뽑기 이력 저장)

### 기능 3: 카드 저장/공유

**설명:** 오늘의 카드 결과를 이미지로 저장하거나 SNS에 공유하여 바이럴 효과를 만든다.

**상세 명세:**
- 카드 결과 화면에 "저장" / "공유" 버튼 배치
- 저장: 카드 이미지 + 카드 이름 + 한 줄 메시지 + 앱 로고를 합성한 이미지를 갤러리에 저장
- 공유: `share_plus` 패키지로 카카오톡/인스타그램 스토리 등 공유
- 공유 이미지 하단에 "매일타로" 워터마크 포함 (브랜딩)
- RepaintBoundary + RenderRepaintBoundary로 위젯을 이미지로 캡처

## 4. 화면별 상세 기획

### 4-1. 스플래시 (SplashScreen)

| 요소 | 내용 |
|------|------|
| UI | 앱 로고 + 앱 이름 "매일타로" + 로딩 인디케이터 |
| 동작 | Hive 초기화 + Supabase 카드 데이터 동기화(최초 1회) + 전면광고 로드 |
| 전환 | 로딩 완료 후 전면광고 표시 -> 홈 화면 이동 |
| 시간 | 최대 3초 (광고 로드 실패 시 바로 홈으로) |

### 4-2. 홈 (HomeScreen) -- 오늘의 카드 뽑기

| 요소 | 내용 |
|------|------|
| 상단 | 날짜 표시 ("3월 26일 수요일") + 인사 메시지 ("오늘 당신에게 필요한 메시지는?") |
| 중앙 | 카드 뒷면 이미지 (탭 유도 텍스트: "카드를 터치하세요") |
| 하단 | 하단 내비게이션 바 (홈 / 도감 / 설정) |
| 동작 | 카드 탭 -> 전면광고 -> 뒤집기 애니메이션 -> 결과 표시 |
| 상태A | 오늘 아직 카드를 뽑지 않은 상태: 카드 뒷면 표시 |
| 상태B | 오늘 이미 카드를 뽑은 상태: 결과 화면 바로 표시 |

### 4-3. 카드 결과 (CardResultScreen)

| 요소 | 내용 |
|------|------|
| 상단 | 카드 이미지 (정방향 또는 역방향) |
| 중단 | 카드 이름 (한글) + 방향 표시 ("정방향" / "역방향") |
| 한 줄 메시지 | 큰 폰트로 강조 (예: "새로운 시작을 두려워하지 마세요") |
| 상세 해석 | 접히는 섹션(ExpansionTile)으로 상세 의미 표시 |
| 버튼 | "저장하기" / "공유하기" 버튼 |
| 하단 | 배너 광고 |

### 4-4. 카드 도감 (CollectionScreen)

| 요소 | 내용 |
|------|------|
| 상단 | 탭바: 전체 / 메이저 / 완드 / 컵 / 소드 / 펜타클 |
| 그리드 | 2열 GridView, 카드 썸네일 + 이름 |
| 표시 | 뽑은 적 있는 카드는 컬러, 없으면 흑백(또는 실루엣) |
| 광고 | 그리드 8번째 아이템마다 네이티브 배너 삽입 |
| 탭 동작 | 카드 상세 화면으로 이동 |

### 4-5. 카드 상세 (CardDetailScreen)

| 요소 | 내용 |
|------|------|
| 상단 | 카드 이미지 (크게) |
| 중단 | 카드 이름 + 번호 |
| 정방향 | 정방향 의미 + 한 줄 메시지 |
| 역방향 | 역방향 의미 + 한 줄 메시지 |
| 롱프레스 | 카드 이미지 또는 해석 영역을 길게 누르면 → 라이더 웨이트(올드스쿨) 원본 이미지 + 전통 해석을 모달로 표시 |
| 하단 | 배너 광고 |

> **올드스쿨 참조 모달**: 커스텀 스킨 사용 시 원본 타로 카드의 디테일(아티팩트 개수, 인물 배치, 상징 요소)을 확인할 수 있도록 라이더 웨이트 이미지를 참조용으로 제공. 올드스쿨 스킨 사용 중일 때는 이미 원본이므로 비활성화.

### 4-6. 설정 (SettingsScreen)

| 요소 | 내용 |

## 5. Supabase 테이블 스키마

### 공통 테이블 (app-factory 공유)

> 상세 스키마: `app-factory/docs/supabase_공통_스키마.md` 참조

| 테이블 | 용도 | 매일타로에서 사용하는 키 |
|--------|------|------------------------|
| apps | 앱 마스터, 활성/비활성, 광고 상태 | `package_name = 'com.pageme.daily_tarot'` |
| admob_ids | AdMob 광고 단위 ID | `package_name` 기준 조회 |
| app_popups | 서버 제어 팝업 (공지, 업데이트 등) | `app_id = 'daily_tarot'` |
| app_privacy | 개인정보처리방침 | `package_name` 기준 조회 |
| app_terms | 이용약관 | `package_name` 기준 조회 |

### 앱 전용 테이블

```sql
-- ============================================
-- 매일타로 전용 스키마 (DDL)
-- ============================================

-- 타로 카드 마스터 데이터
CREATE TABLE tarot_cards (
  id SERIAL PRIMARY KEY,
  name TEXT NOT NULL,               -- 영문명: "The Fool", "The Magician"
  name_kr TEXT NOT NULL,            -- 한글명: "바보", "마법사"
  arcana TEXT NOT NULL,             -- "major" / "minor"
  suit TEXT,                        -- NULL(메이저) / "wands" / "cups" / "swords" / "pentacles"
  number INT NOT NULL,              -- 메이저: 0~21, 마이너: 1~14(Ace~King)
  image_url TEXT,                   -- Supabase Storage 이미지 경로
  upright_meaning TEXT NOT NULL,    -- 정방향 상세 의미
  reversed_meaning TEXT NOT NULL,   -- 역방향 상세 의미
  upright_message TEXT NOT NULL,    -- 정방향 한 줄 메시지
  reversed_message TEXT NOT NULL,   -- 역방향 한 줄 메시지
  app_id TEXT NOT NULL DEFAULT 'daily_tarot',
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX idx_tarot_cards_app_id ON tarot_cards(app_id);
CREATE INDEX idx_tarot_cards_arcana ON tarot_cards(arcana);

ALTER TABLE tarot_cards ENABLE ROW LEVEL SECURITY;

CREATE POLICY "tarot_cards_read_all" ON tarot_cards
  FOR SELECT USING (true);
```

### Supabase Storage

- 버킷명: `card-images`
- 폴더 구조: `card-images/daily_tarot/{skin_id}/{card_id}.png`
- 기본 스킨: `skin_id = "default"`
- 올드스쿨 참조: `skin_id = "rider_waite"` (라이더 웨이트 원본, 모달 참조용)
- 공개 읽기 허용

**데이터 시드 참고:**
- 메이저 아르카나: 22장 (0번 The Fool ~ 21번 The World)
- 마이너 아르카나: 56장 (4수트 x 14장)
- 총 78장
- 초기 데이터는 SQL INSERT 또는 Supabase Dashboard CSV 업로드로 투입

### 로컬 에셋 (앱 번들)

| 경로 | 내용 | 비고 |
|------|------|------|
| `assets/images/cards/default/` | 기본 스킨 카드 이미지 78장 | 미드저니 생성, 앱 번들에 포함 |
| `assets/images/cards/rider-waite/` | 라이더 웨이트 원본 78장 | 퍼블릭 도메인, 올드스쿨 참조 모달용 |
| `assets/images/cards/card_back.png` | 카드 뒷면 디자인 1장 | 모든 스킨 공통 또는 스킨별 분리 |

## 6. AdMob 광고 배치 전략

| 위치 | 광고 유형 | 타이밍 | eCPM 기대 |
|------|----------|--------|-----------|
| 스플래시 후 | 전면광고 (Interstitial) | 앱 실행 시 1회 | 높음 |

| 카드 상세 화면 하단 | 배너 광고 (Banner) | 상세 열람 중 상시 | 중간 |

**광고 정책:**
- 전면광고는 하루 최대 2회 노출 (스플래시 1회 + 카드 뽑기 1회)

- 테스트 빌드에서는 반드시 테스트 광고 ID 사용
- 프로덕션 광고 ID는 `.env` 파일에서 환경변수로 관리

## 7. 디자인 가이드라인

### 컨셉: "Calm Mystical" -- 차분하면서 신비로운

기존 타로 앱들의 어둡고 무거운 톤 대신, 밝고 여백 많은 미니멀 디자인. 단, 타로 특유의 신비로운 느낌은 유지.

### 컬러 팔레트

| 용도 | 컬러 | HEX | 설명 |
|------|------|-----|------|
| Primary | 소프트 라벤더 | `#B8A9E8` | 메인 액센트, 버튼 |
| Primary Dark | 딥 퍼플 | `#7C6BB5` | 헤더, 강조 텍스트 |
| Secondary | 소프트 골드 | `#E8D5A0` | 카드 테두리, 포인트 |
| Background | 크림 화이트 | `#FAFAF7` | 라이트 모드 배경 |
| Background Dark | 미드나잇 블루 | `#1A1A2E` | 다크 모드 배경 |
| Surface | 화이트 | `#FFFFFF` | 카드, 시트 배경 |
| Text Primary | 차콜 | `#2D2D3A` | 본문 텍스트 |
| Text Secondary | 그레이 | `#8E8E9A` | 보조 텍스트 |

### 타이포그래피

| 용도 | 서체 | 크기 | 가중치 |
|------|------|------|--------|
| 카드 이름 | Pretendard (또는 Noto Sans KR) | 24sp | Bold |
| 한 줄 메시지 | Pretendard | 20sp | Medium |
| 상세 해석 | Pretendard | 16sp | Regular |
| 날짜/보조 텍스트 | Pretendard | 14sp | Regular |

### 디자인 원칙

1. **여백 우선**: 콘텐츠 사이 최소 16dp 간격, 카드 주변은 24dp 이상
2. **부드러운 모서리**: 카드 radius 16dp, 버튼 radius 12dp
3. **미니멀 애니메이션**: 카드 뒤집기 외에는 페이드 인/아웃 정도만
4. **아이콘**: 라인 스타일 아이콘 (Lucide Icons 또는 Material Symbols Outlined)

## 9. ASO 키워드 제안

### 앱 이름/부제 키워드

| 위치 | 텍스트 |
|------|--------|
| 앱 이름 | 매일타로 - 오늘의 카드 한 장 |
| 짧은 설명 | 매일 아침, 나를 위한 타로 메시지 한 장. 감성 타로 카드 뽑기 |

### 타겟 키워드 (한국어)

**주력 키워드:** 타로, 오늘의 타로, 데일리 타로, 타로카드, 타로 운세
**니치 키워드:** 감성 타로, 타로 한장, 오늘의 카드, 하루 타로, 미니멀 타로
**연관 키워드:** 운세, 오늘의 운세, 카드 뽑기, 타로 리딩, 셀프 타로

### ASO 전략

1. 앱 이름에 "타로" + "오늘의 카드" 키워드 삽입
2. 스크린샷에 밝고 미니멀한 디자인을 강조하여 경쟁 앱과 시각적 차별화
3. 짧은 설명에 "감성", "미니멀", "매일" 키워드 포함
4. 긴 설명 첫 줄에 핵심 키워드 배치

- 스킨 상점 UI에 "잠긴 스킨" 미리보기 표시 (기대감 형성, v1.0에서 선택적)
