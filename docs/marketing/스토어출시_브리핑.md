### 1-1. UI/UX 관점 점검

#### 스크린샷 촬영 적합성

**잘 나오는 요소 (강점)**

- 카드 결과 화면: 파스텔 미니멀 카드 이미지 + 큰 폰트 한 줄 메시지 조합이 스크린샷 한 장에 모든 것을 담을 수 있어 매우 유리하다.
- 카드 도감 그리드: 78장 썸네일이 깔리는 2열 그리드는 "콘텐츠 풍부함"을 시각적으로 전달하기 좋다. 컬러/흑백 대비 UI(뽑은 카드=컬러, 미뽑은=흑백)는 수집 욕구를 자극하는 그림이 된다.
- 홈 화면(카드 뒷면): 여백 중심으로 카드 한 장이 크게 배치되면 앱의 미니멀 컨셉을 직관적으로 전달한다.

**개선이 필요한 요소 (개발 단계에서 반영 권장)**

- 한 줄 메시지 텍스트 크기: 스크린샷에서 가독성이 떨어지지 않도록 최소 20sp 이상을 유지해야 한다. 소폰트 스크린샷은 Play Store 썸네일(작게 보일 때) 에서 인상을 남기지 못한다.
- 배너 광고 노출 상태: 스크린샷은 반드시 광고 없는 상태에서 촬영해야 한다. 개발 단계에서 "스크린샷 모드"(광고 숨김 플래그)를 debug 옵션으로 구현해두면 편리하다.
- 카드 상세 화면의 ExpansionTile: 접힌 상태에서 스크린샷을 찍으면 내용이 보이지 않는다. 스크린샷 촬영 시에는 펼친 상태로 찍거나, 별도 디자인 카피 화면을 만드는 방향을 고려한다.
- 다크 모드 스크린샷 별도 준비: Play Store는 다크 모드 스크린샷 세트를 별도로 올릴 수 있다. 라이트(낮) / 다크(밤) 두 세트 준비 시 미드나잇 블루 배경의 다크 버전이 타로 앱의 신비감을 훨씬 강화한다.

#### 첫 실행 경험(FTUE) 리텐션 관점

**현재 플로우 분석**

```
앱 실행 → 스플래시(최대 3초, 전면광고) → 홈(카드 뒷면) → 카드 탭 → 전면광고 → 뒤집기 → 결과
```

**문제 포인트**

1. 최초 실행 시 전면광고를 두 번 연속 마주치는 구조다(스플래시 후 + 카드 탭 후). 신규 사용자가 앱을 열자마자 광고를 두 번 보면 삭제 이탈 확률이 높아진다.
   - **권장 조치**: 최초 설치 후 첫 실행에 한해 스플래시 전면광고를 건너뛴다. Hive에 `isFirstLaunch` 플래그를 저장하여 처리. 카드 탭 후 광고는 유지(기대감이 형성된 시점이므로 덜 거슬림).

2. 스플래시에서 Supabase 동기화 실패 시 바로 홈으로 넘어가면 카드 데이터가 없을 수 있다.
   - **권장 조치**: 카드 데이터는 앱 번들 로컬 에셋(`assets/images/cards/default/`)으로 오프라인에서도 동작해야 한다. Supabase는 텍스트 데이터(의미, 메시지) 동기화 용도로만 사용하되, 로컬 fallback JSON을 번들에 포함하는 것을 권장한다.

3. 알림 권한 요청 타이밍이 PRD에 명시되어 있지 않다.
   - **권장 조치**: 첫 실행 직후(스플래시나 홈 직접) 요청하면 수락률이 낮다. 카드 결과를 한 번 확인한 직후, "내일도 오늘 같은 메시지를 받아보시겠어요?" 식의 컨텍스트를 제공한 뒤 알림 권한을 요청하면 수락률이 훨씬 높아진다.

#### 공유 이미지 바이럴 최적화

**현재 설계**: 카드 이미지 + 카드 이름 + 한 줄 메시지 + 앱 로고 합성 이미지

**바이럴 최적화 권장 사항**

- 이미지 비율: 인스타그램 스토리(9:16) 비율을 기본으로 한다. 정사각형(1:1)보다 세로형이 공유 시 피드를 꽉 채워 눈에 잘 띈다.
- 한 줄 메시지 폰트를 공유 이미지에서 더 크게: 카카오톡 공유 시 썸네일이 작게 보이므로, 메시지 텍스트가 썸네일에서도 읽힐 정도로 커야 한다.
- 날짜 포함: 공유 이미지 하단에 "2026년 4월 10일의 카드" 식으로 날짜를 표시하면 매일 다른 카드를 뽑는 동기가 강화되고, SNS 피드에서 시의성이 생긴다.
- 워터마크 위치: 하단 중앙보다는 하단 우측 코너에 소형으로 배치하는 것이 이미지를 덜 가린다. 크기는 카드 이미지 폭의 15% 이내를 권장한다.
- 배경 블러 또는 그라디언트: 카드 이미지 주변에 소프트 라벤더 그라디언트 배경을 넣으면 공유 이미지가 완성형 비주얼처럼 보인다.

### 1-2. 스크린샷 화면 후보 (5~8장)

스크린샷은 Play Store 기준 최대 8장, App Store는 10장까지 등록 가능하다.
**스토어에서 처음 보이는 3장이 전환율의 70% 이상을 결정**한다. 앞 3장에 핵심 가치를 집중한다.

| 순서 | 화면 | 카피 제안 | 목적 |
|------|------|----------|------|
| 1 | 카드 결과 화면 (대표 카드 + 임팩트 있는 한 줄 메시지) | "매일 아침, 나에게 필요한 한 마디" | 앱의 핵심 경험 첫 인상 |
| 2 | 홈 화면 (카드 뒷면 + 탭 유도) | "오늘의 카드를 뽑아보세요" | 사용 방법이 직관적임을 전달 |
| 3 | 공유 이미지 (실제 공유 결과물 예시) | "친구에게 공유하고 함께 타로 봐요" | 바이럴 가능성 + SNS 연결 |
| 4 | 카드 도감 (그리드, 컬러/흑백 대비) | "78장 카드를 하나씩 수집하세요" | 수집 게임성, 재방문 유인 |
| 5 | 카드 상세 (카드 이미지 크게 + 정/역방향 의미 펼침) | "정방향과 역방향, 두 가지 메시지" | 콘텐츠 깊이감 전달 |
| 6 | 카드 결과 다크 모드 버전 | "밤에는 다크 모드로 조용히" | 다크 모드 지원 어필, 감성 강화 |
| 7 | 설정 화면 알림 설정 | "매일 아침 타로 알림, 루틴으로 만들어요" | 리텐션 기능(알림) 어필 |
| 8 | 카드 상세 롱프레스 모달 (RW 원본 참조) | "타로 원본도 함께 볼 수 있어요" | 차별화 기능, 타로 팬 타겟 |

**스크린샷 제작 시 공통 가이드**

- 상단 카피: 흰색 또는 딥 퍼플(#7C6BB5) 볼드 텍스트, 배경과 대비 확보
- 하단 부제: 소프트 라벤더 계열, 레귤러 웨이트
- 기기 프레임: 픽셀 또는 아이폰 프레임 사용(과도한 장식 없이 심플하게)
- 배경: 크림 화이트(#FAFAF7) 또는 소프트 라벤더 그라디언트 통일
- 해상도: 1080x1920px(세로) 기준

### 1-3. 사용 흐름 분석

#### 정상 플로우

```
앱 실행
  └─ 스플래시(최대 3초)
       └─ 전면광고(스플래시 후)
            └─ 홈 화면
                 ├─ [오늘 미뽑기] 카드 뒷면 → 탭 → 전면광고 → 뒤집기(0.8초) → 결과 화면
                 │     └─ 저장 / 공유 / 배너 광고 상시 노출
                 ├─ [오늘 이미 뽑기] 결과 화면 바로 표시
                 └─ 도감 / 설정 탭 이동
```

#### 이탈 포인트 및 대응

| 이탈 포인트 | 위험도 | 대응 방안 |
|-----------|--------|----------|
| 스플래시 전면광고 (최초 실행 시) | 높음 | 첫 실행은 스플래시 광고 건너뜀(isFirstLaunch 처리) |
| 카드 탭 직후 전면광고 | 중간 | 광고 로드 실패 시 즉시 뒤집기로 넘어가도록 타임아웃(2초) 설정 필수 |
| Supabase 동기화 실패/지연 | 중간 | 로컬 번들 fallback 데이터로 오프라인 동작 보장 |
| 도감 그리드 8개마다 네이티브 광고 | 중간 | 광고 로드 실패 시 해당 슬롯을 빈 공간이 아닌 다음 카드로 채움 |
| 결과 화면의 ExpansionTile 미발견 | 낮음 | "더 보기" 텍스트나 화살표 아이콘으로 존재를 명시적으로 알림 |

#### 리텐션 훅 점검

| 리텐션 메커니즘 | 현재 설계 여부 | 권장 강화 방안 |
|--------------|-------------|--------------|
| 매일 알림 | 있음(설정에서 ON/OFF) | FTUE 후 맥락 기반 권한 요청으로 수락률 향상 |
| 도감 수집 진행률 표시 | 간접적(컬러/흑백 대비) | 홈 화면 또는 도감 탭에 "78장 중 N장 수집" 배지 추가 권장 |
| 연속 출석 스트릭 | 없음 | 도감 화면 상단에 "N일 연속 뽑기" 표시(개발 공수 낮음, 리텐션 효과 높음) |
| 파생 앱 교차 홍보 | 있음(설정 화면) | 앱 팝업(`app_popups`)으로 특정 시점에 노출하는 것이 설정 화면보다 효과적 |

### 1-4. 스토어 등록 필수 요소

#### 앱 아이콘 방향

**디자인 컨셉**: 소프트 라벤더 원형 배경 + 중앙에 단일 타로 카드 모티프

**구체적 묘사**:
- 배경: 소프트 라벤더(#B8A9E8) → 딥 퍼플(#7C6BB5) 원형 그라디언트
- 중앙 모티프: 별 문양이 있는 미니멀 카드 뒷면 일러스트 (카드 테두리는 소프트 골드 #E8D5A0)
- 또는: 메이저 아르카나 "별(The Star)" 카드의 별 심볼 단독 추출 (별 = 희망/일일 루틴의 상징)
- 폰트: 아이콘 내에 텍스트 최소화(Play Store 아이콘은 작게 보일 때 텍스트 가독성 제로)
- 스타일: 라운드 직사각형(모서리 반경 넉넉히), 앱 번들의 미니멀 파스텔 카드 스타일과 일관성 유지
- 피해야 할 것: 펜타그램(오각별 마법진), 해골/악마 이미지는 스토어 심사에서 불이익 가능

**"매일" 시리즈 아이콘 체계**:
- 매일타로: 라벤더 배경 + 카드 심볼
- 매일오라클: 소프트 핑크 배경 + 구체/크리스털 심볼
- 매일룬: 스톤 그레이 배경 + 룬 문자 심볼

#### 피처 그래픽 (1024x500) 방향

- 배경: 크림 화이트(#FAFAF7) 또는 소프트 라벤더 → 화이트 그라디언트
- 좌측: 대표 타로 카드 이미지 2~3장을 살짝 겹쳐서 사선으로 배치 (생동감)
- 우측: 앱 이름 "매일타로" (딥 퍼플, 볼드) + 한 줄 태그라인 (소프트 골드 또는 그레이)
- 태그라인 후보: "매일 아침, 나를 위한 한 장" 또는 "타로계의 명언 앱"
- 전체 톤: 밝고 따뜻하게. 경쟁 앱의 어둡고 신비로운 피처 그래픽과 반대 방향으로 차별화

#### 개인정보처리방침

**필요 여부: 필수**

- AdMob 광고 게재 앱은 개인정보처리방침이 Google Play 정책상 의무다.
- Firebase Crashlytics 사용 시도 마찬가지로 의무다.
- 앱 내 개인정보처리방침 링크: PRD 설정 화면에 이미 포함되어 있음 (app_privacy 테이블에서 서버 제어).
- 스토어 등록 정보의 "개인정보처리방침 URL" 필드에도 동일 URL을 입력해야 한다.
- **호스팅 방법 권장**: Supabase Edge Function 또는 단순 GitHub Pages 정적 HTML로 호스팅. 별도 도메인 불필요.

#### 콘텐츠 등급 관련 주의사항

| 항목 | 내용 |
|------|------|
| Google Play 등급 | 만 3세 이상(Everyone)으로 신청 가능. 타로/운세는 금지 카테고리 아님. |
| 앱 카테고리 | "라이프스타일" 또는 "엔터테인먼트" 권장. "점술/운세" 세부 카테고리는 별도 존재하지 않음. |
| 면책 문구 | PRD에 명시된 "본 앱은 엔터테인먼트 목적으로 제공됩니다" 문구를 앱 내(설정 화면, 스플래시)와 스토어 설명 하단 모두에 포함할 것. |
| App Store (애플) | 애플은 점술/운세 앱에 더 까다롭다. "Entertainment" 카테고리로 신청하고, 설명에 "오락/재미 목적"을 명시하면 심사 통과 가능성 높음. "점을 봐드립니다" 식의 과장 표현은 피한다. |
| 역방향 카드 이미지 | 180도 회전된 이미지 자체는 심사 문제 없음. 단, 악마/해골 계열 카드 이미지가 앱 스크린샷에 전면으로 노출되면 심사자 재량에 따라 문제가 될 수 있으니 스크린샷용 카드는 긍정적인 카드를 선택한다. |
| 광고 등급 | AdMob에서 타겟 연령을 "일반"으로 설정. "아동용 앱"으로 설정하지 않는다(타로는 아동 대상이 아님). |

## Part 2. 앱 브랜딩 브리핑

---

### 2-1. 브랜드 아이덴티티

#### 톤앤매너

**한 문장 정의**: 당신의 오늘을 가볍게 열어주는, 따뜻하고 차분한 친구.

| 구분 | 방향 |
|------|------|
| 말투 | 존댓말, 단 딱딱하지 않은 부드러운 경어체. "~하세요"보다 "~해보세요", "~어떨까요?" |

### 2-2. 비주얼 아이덴티티

#### 앱 아이콘 디자인 방향 (구체적 묘사)

**권장안 A — "빛나는 카드 뒷면"**

배경은 소프트 라벤더(#B8A9E8)에서 딥 퍼플(#7C6BB5)로 이어지는 원형 그라디언트. 중앙에 세로형 카드를 약 10도 살짝 기울여 배치하되, 카드 뒷면에 별 문양(소프트 골드 라인)이 들어간 디자인. 카드 테두리는 소프트 골드. 배경과 카드 사이에 은은한 광원(glow) 효과로 신비감 추가.

**권장안 B — "별 하나"**

배경은 위와 동일(라벤더 → 딥 퍼플 원형). 카드 없이 중앙에 4~5각 별 심볼 하나만 소프트 골드로. 극도로 미니멀하여 앱 그리드에서 깔끔하게 보임. 별은 "The Star(별)" 카드의 상징이자 매일(daily)의 상징으로 이중 의미.

**비추천**: 사람 얼굴/인물, 복잡한 패턴, 어두운 배경. Play Store 앱 그리드에서 작게 보일 때 인식이 어렵다.

#### 스플래시 화면 디자인 방향

- 배경: 크림 화이트(#FAFAF7), 중앙 정렬
- 상단 1/3: 앱 아이콘 (72~96dp 크기)
- 중앙: 앱 이름 "매일타로" (딥 퍼플, Pretendard Bold, 28~32sp)
- 하단: "오늘의 카드를 불러오는 중..." (그레이, 14sp) + 점 애니메이션 인디케이터
- 전체 분위기: 매우 심플. 로딩 자체가 짧아야 하므로(최대 3초) 화려한 애니메이션은 지양.
- 다크 모드: 배경이 미드나잇 블루(#1A1A2E)로 전환, 텍스트는 크림 화이트.

**공유 이미지 레이아웃 (세로 9:16 기준, 1080x1920px)**

